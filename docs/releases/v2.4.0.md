# Release v2.4.0

## Tổng quan

v2.4.0 là bản ổn định sau các phase lớn:

- GEDCOM Import/Merge v2.3.4
- Data Maintenance v2.3.5
- Admin Health v2.3.6
- Vietnamese Tree Hardening v2.3.7

Mục tiêu của bản này là khóa lại hệ thống genealogy sau khi chuyển sang Family Model + Event Model, bổ sung GEDCOM staging an toàn, công cụ bảo trì dữ liệu, health dashboard và diagnostics cho cây gia phả tiếng Việt.

---

## 1. GEDCOM Import/Merge

### Hoàn thành

- GEDCOM export dùng stable UUID XREF.
- GEDCOM parser giữ UUID XREF khi import lại.
- Staging import trước khi ghi DB chính.
- Duplicate detection cho person.
- Match Review cho person trùng/nghi trùng.
- Round-trip validation report.
- Merge Plan preview-only.
- Merge Suggestions review layer.
- Commit approved merge suggestions bằng RPC.
- Post-import audit report.
- Import Safety Gate.
- GEDCOM Import/Merge runbook.


## 8. Final SQL audit

Chạy trước khi tag release:

```sql
SELECT COUNT(*) AS active_unknown_left
FROM public.persons
WHERE full_name IN ('Unknown', 'Chưa rõ tên')
  AND deleted_at IS NULL;

SELECT COUNT(*) AS events_without_person_events
FROM public.events e
WHERE e.deleted_at IS NULL
  AND e.legacy_person_id IS NOT NULL
  AND e.type IN ('birth', 'death')
  AND NOT EXISTS (
    SELECT 1 FROM public.person_events pe
    WHERE pe.event_id = e.id
      AND pe.person_id = e.legacy_person_id
  );

WITH grouped AS (
  SELECT
    e.legacy_person_id,
    e.type,
    e.start_date,
    e.sort_date,
    COUNT(*) AS c
  FROM public.events e
  WHERE e.deleted_at IS NULL
    AND e.legacy_person_id IS NOT NULL
    AND e.type IN ('birth', 'death')
  GROUP BY e.legacy_person_id, e.type, e.start_date, e.sort_date
  HAVING COUNT(*) > 1
)
SELECT COUNT(*) AS duplicate_groups
FROM grouped;

SELECT COUNT(*) AS active_empty_families
FROM public.families f
WHERE f.deleted_at IS NULL
  AND NOT EXISTS (
    SELECT 1 FROM public.family_parents fp
    WHERE fp.family_id = f.id
  )
  AND NOT EXISTS (
    SELECT 1 FROM public.family_children fc
    WHERE fc.family_id = f.id
  );

SELECT COUNT(*) AS approved_merge_suggestions
FROM public.import_merge_suggestions
WHERE status = 'approved';

SELECT COUNT(*) AS pending_merge_suggestions
FROM public.import_merge_suggestions
WHERE status = 'pending';
```

Kỳ vọng tốt:

```text
active_unknown_left = 0
events_without_person_events = 0
duplicate_groups = 0
active_empty_families = 0
approved_merge_suggestions = 0 hoặc đã hiểu rõ
pending_merge_suggestions = 0 hoặc đang review
```

## 10. Production runtime

Production hiện chạy bằng root PM2:

```text
name: giapha
cwd: /opt/giapha-os
command: bun run start
port: 3000
```

Kiểm tra:

```bash
sudo pm2 list
sudo pm2 describe giapha
sudo ss -ltnp | grep ':3000'
curl -I http://127.0.0.1:3000
```

Restart chuẩn:

```bash
cd /opt/giapha-os

git checkout main
git pull origin main

bun install
bun run release:check

sudo rm -rf .next
bun run build

sudo pm2 restart giapha --update-env
sudo pm2 save
```

Nếu PM2 báo `bun: command not found`, start bằng absolute bun path hoặc tạo symlink `/usr/local/bin/bun`.

## 14. Khi nào bỏ legacy

Chưa bỏ legacy ngay ở v2.4.0.

Chỉ nên bỏ khi đạt đủ:

```text
1. Tất cả UI đọc quan hệ từ Family Model / Event Model.
2. Tất cả action thêm/sửa/xóa quan hệ ghi vào Family Model / Event Model.
3. Legacy relationships/person date fields chỉ còn để audit, không còn là source of truth.
4. GEDCOM import/export dùng model mới hoàn toàn.
5. Audit chạy sạch trong nhiều vòng sử dụng thật.
```

Lộ trình đề xuất:

```text
v2.4.2: Sửa RelationshipManager đọc/ghi Family Model đầy đủ.
v2.4.3: Repair legacy marriages/parent-child còn thiếu Family Model.
v2.4.4: Legacy Compatibility Audit page.
v2.5.0: Model mới là source of truth, legacy chỉ readonly fallback.
v2.6.0: Archive legacy relationships nếu audit sạch lâu dài.
```
